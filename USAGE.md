## 1. 平台简介

涌金阁是一个支持 **A股 / 港股 / 美股** 三大市场的月K线量化分析平台，核心功能包括：

- 按月份统计历史涨跌概率（最早追溯至 2000 年）
- 月份筛选：找出在特定月份历史胜率最高的股票
- 行业分析：从行业维度筛选季节性规律
- 多数据源支持，数据可离线存储，查询极速

## 2. 安装与启动

### 桌面版（推荐）

1. 运行 `涌金阁_Setup_v1.0.0.exe`，按提示完成安装
2. 安装完成后系统会自动注册为 **Windows 服务**，开机自启
3. 打开浏览器访问 **http://localhost:8588**
4. 首次访问使用默认管理员账号登录：
   - 用户名：`admin`
   - 密码：`admin123`

> **提示**：安装时会自动安装所有依赖包（首次安装约需 3-5 分钟），请耐心等待进度条完成。

### 开发模式（命令行）

```bash
pip install -r requirements.txt
python start_prod.py
```

## 3. 首次配置

登录后进入 **数据管理** 页面，完成以下配置：

### 3.1 配置数据源

在"数据源配置"区域，分别为三个市场选择数据源（详见 [第4节](#4-数据源说明)）：

| 市场 | 推荐数据源 | 备注 |
|------|-----------|------|
| 🇨🇳 A股 | **akshare** | 免费，无需配置 |
| 🇭🇰 港股 | **akshare** | 免费，覆盖全量 2700+ 只 |
| 🇺🇸 美股 | **yfinance** | 免费，覆盖主流美股 |

配置完成后点击 **保存配置**。

### 3.2 首次数据更新

1. 在"数据更新操作"区域选择更新市场（建议先单独更新 A股，再更新港股/美股）
2. 点击 **增量更新**
3. 等待进度条完成

> ⚠️ 首次全量更新耗时：A股约 20 分钟，港股约 30 分钟，美股约 40 分钟（取决于网络环境）。

## 4. 数据源说明

### 4.1 A股数据源

| 数据源 | 费用 | 说明 |
|--------|------|------|
| **akshare**（推荐） | 免费 | 东方财富接口，数据完整，无需注册 |
| **BaoStock** | 免费 | 证券宝接口，稳定，北交所数据自动降级至 akshare |
| **Tushare Pro** | 需积分 | 数据质量高，需在 [tushare.pro](https://tushare.pro) 注册并填入 Token |
| **JQData** | 需账号 | 聚宽数据，需填入手机号和密码 |

### 4.2 港股数据源

| 数据源 | 费用 | 覆盖只数 | 说明 |
|--------|------|----------|------|
| **akshare**（推荐） | 免费 | 全量 2700+ 只 | 东财接口，`stock_hk_hist`，支持前复权，约 30 分钟全量 |
| **yfinance** | 免费 | 约 900 只 | 仅覆盖知名蓝筹，小型港股无数据 |
| **Tushare Pro** | 需积分 | 2000+ 只 | 需 Tushare Token，日K线聚合为月K |

> **建议**：港股优先使用 akshare，可覆盖所有在市港股，而 yfinance 只有约 33% 覆盖率。

### 4.3 美股数据源

| 数据源 | 费用 | 覆盖只数 | 更新速度 | 说明 |
|--------|------|----------|----------|------|
| **yfinance**（推荐） | 免费 | 8000+ 只 | 批量下载，约 40 分钟 | 支持复权，批量接口较快 |
| **Alpha Vantage** | 需 API Key | 8000+ 只 | 串行，Premium 约 107 分钟 | 复权收盘价（`adjusted close`），免费版仅 25 次/天 |
| **Tushare Pro** | 需积分 | 10000+ 只 | 串行 | 股票列表来自 Tushare，K线仍使用 yfinance |

#### Alpha Vantage 配置说明

1. 在 [alphavantage.co](https://www.alphavantage.co/support/#api-key) 申请免费 API Key
2. 在"数据源配置 → 美股"选择 **Alpha Vantage**，填入 API Key
3. **免费版限制**：25 次/天，仅适合测试单只股票，不适合全量更新
4. **Premium 版**（$49.99/月）：75 次/分钟，可完成全量更新
5. Premium 用户需在 `config.json` 中设置：
   ```json
   "alpha_vantage": {
     "api_key": "你的Key",
     "requests_per_minute": 75
   }
   ```

## 5. 数据管理

进入首页 → **数据管理** 标签页。

### 5.1 更新模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **增量更新** | 只补充上次更新后缺少的数据 | 日常每月执行一次 |
| **全量重建** | 先清空当前数据源数据，再从头重新获取 | 切换数据源后、数据异常时 |
| **测试更新** | 只更新前 10 只股票，验证配置是否正确 | 首次配置后验证 |

### 5.2 更新市场选择

- **全部**：同时更新 A股 + 港股 + 美股
- **A股 / 港股 / 美股**：单独更新指定市场

### 5.3 断点续更

更新过程中若中途停止（网络中断、手动停止），下次更新时勾选 **从断点继续**，可从中断处继续，不会重复下载已完成的股票。

### 5.4 行业分类更新

行业分类不包含在常规 K 线更新中，需单独点击 **更新行业分类**：
- A股：申万行业 / 中信行业（来自 Tushare/akshare）
- 港股/美股：通过 yfinance 逐只获取 sector 信息（耗时较长）

### 5.5 数据状态查看

页面上方"数据概览"显示：
- 各市场股票总数、有K线数量、覆盖率
- 各数据源记录数、时间跨度

## 6. 功能模块详解

### 6.1 单只股票分析

**路径**：首页 → 单只股票分析

**操作步骤**：
1. 输入股票代码：
   - A股：`000001`（平安银行）
   - 港股：`00700`（腾讯）或 `00700.HK`
   - 美股：`AAPL` 或 `AAPL.US`
2. 选择分析类型：
   - **单月统计**：指定某一月份（如每年1月），统计历史胜率
   - **多月份统计**：同时查看全年各月份的胜率对比
3. 设置年份范围（如 2010-2025）
4. 点击查询

**结果说明**：
- **上涨概率**：历史上该月份收阳（月末收盘 > 月初开盘）的占比
- **平均涨幅 / 平均跌幅**：上涨年份 / 下跌年份的平均百分比
- **数据源**：显示本次查询使用的数据源

### 6.2 月份筛选统计

**路径**：首页 → 月份筛选

**功能**：在指定月份和年份范围内，找出历史胜率最高的前 N 只股票。

**操作步骤**：
1. 选择月份（1-12月）
2. 设置年份范围
3. 选择市场（A股 / 港股 / 美股 / 全部）
4. 设置最少数据年数（过滤掉上市时间太短、样本不足的股票）
5. 点击查询

**增强分析**：勾选后额外显示：
- **期望收益率**：上涨概率 × 平均涨幅 − 下跌概率 × 平均跌幅
- **近5年一致性**：近5个样本是否均上涨（稳定性指标）
- **跑赢大盘概率**：相对于大盘指数的超额收益概率

### 6.3 行业分析

**路径**：首页 → 行业分析

#### 行业统计

1. 选择行业分类：**申万行业**（A股）或 **中信行业**
2. 选择市场、月份、年份范围
3. 查看各行业按上涨概率排名

#### 行业增强分析

与月份筛选增强分析类似，以行业为单位输出期望收益率、一致性等指标。

#### 行业前N支股票

1. 先执行行业统计，点击某个行业名称
2. 查看该行业内胜率最高的前 N 只个股

### 6.4 LOF 基金套利

**路径**：首页 → LOF套利

功能包括：
- 实时查看 LOF 基金折溢价率
- 记录套利操作（买入/卖出）
- 设置折溢价提醒阈值

### 6.5 数据源对比

**路径**：数据管理 → 数据源对比

在同一股票上对比不同数据源（如 tushare vs akshare）的 K 线差异，适合验证数据质量。

### 6.6 Excel 导出

所有分析结果页面右上角均有 **导出 Excel** 按钮，导出当前查询结果为 `.xlsx` 文件。

## 8. 常见问题

**Q：港股更新后显示覆盖率只有 33%，其余股票无K线？**

A：默认数据源 yfinance 只收录约 900 只知名港股。请在"数据源配置"将港股改为 **akshare**，再执行全量重建，即可覆盖全部 2700+ 只港股（约 30 分钟）。

---

**Q：数据更新中途断网/关机，下次如何继续？**

A：更新时系统每完成一批就保存断点。下次进入数据管理页，点击 **从断点继续**，程序会跳过已完成的股票，从中断处恢复。

---

**Q：如何获取 Tushare Token？**

A：访问 [tushare.pro](https://tushare.pro) 注册账号，在个人中心可获取 Token。A股使用 Tushare 需要约 120 积分（每月免费赠送），港股日线（`hk_daily`）需要约 2000 积分。

---

**Q：Alpha Vantage 免费 API Key 能完成全量美股更新吗？**

A：不能。免费版限制 25 次/天，全量 8000 只美股需要 320 天。建议：
- 日常使用 **yfinance**（免费，批量下载，40 分钟完成）
- 需要 Alpha Vantage 复权数据时，升级至 Premium 版（$49.99/月）并设置 `requests_per_minute: 75`

---

**Q：行业分类为空怎么办？**

A：行业分类需单独更新，在数据管理页点击 **更新行业分类** 按钮。A股行业分类更新较快（约 1 分钟），港股/美股通过 yfinance 逐只获取，耗时较长（数百只约 30 分钟）。

---

**Q：查询结果显示"暂无数据"但数据管理里显示有K线？**

A：可能原因：
1. 查询的年份范围内该股票尚未上市（缩小年份范围）
2. 数据源不匹配——查询时使用的数据源与更新时的数据源不同（检查当前数据源配置）
3. 该股票本身在该月份没有历史样本（样本太少）

---

**Q：切换了数据源后，旧数据源的K线还在吗？**

A：在。系统按 `(股票代码, 日期, 数据源)` 三元组存储，多个数据源的数据可以共存。执行全量重建只会清除**当前配置数据源**的数据，不影响其他数据源。可通过"数据源对比"功能查看各数据源差异。

---

**Q：如何彻底重置数据库？**

A：删除安装目录下的 `stock_data.db` 文件后重启服务，系统会自动创建空数据库，再重新执行全量更新即可。
